## ✨ Overview

Game-AI-Assignment-3 is a comprehensive Unity-based toolkit designed to streamline asset integration, animation, and scene optimization for immersive game development. 

**Why Game-AI-Assignment-3?**

This project simplifies complex asset and scene management while ensuring consistent animation and behavior across your game. The core features include:

- 🧩 **🎯 Puzzle Piece:** Dependency management for Unity packages, ensuring seamless integration of modules like AI navigation, physics, and rendering.
- 🚀 **🌟 Highlight:** Reusable prefab assets and detailed setup guides for characters and environments.
- 🎬 **🎥 Spotlight:** Advanced animation controllers and clips that bring characters and objects to life.
- ⚙️ **🔧 Gear:** Scene optimization through occlusion culling settings for smooth performance.
- 💡 **✨ Spark:** Modular design supporting rapid prototyping, scene assembly, and scalable development.

## 📌 Features

| Feature Category             | Description                                                                                     |
|------------------------------|-------------------------------------------------------------------------------------------------|
| **Programming Language**     | Unity C# (implied by Unity-specific assets and controllers)                                    |
| **Game Engine**              | Unity Engine                                                                                    |
| **Asset Types**              | - 3D Models: FBX files (e.g., `sj001.fbx`, `zombie crawl.fbx`, `polygonal metalon.fbx`)       |
|                              | - Prefabs: `.prefab` (e.g., `rabbit 1.prefab`, `sphere.prefab`, `turtleshell.prefab`)            |
|                              | - Animations: `.anim`, `.fbx` (e.g., `attack01_slime_anim.fbx`, `jumpingup.anim`)             |
|                              | - Controllers: `.controller` (e.g., `characteranim.controller`)                              |
|                              | - Textures & Shaders: `.psd`, `.tga`, `.exr` (e.g., `boximon fiery.psd`, `reflectionprobe-0.exr`) |
|                              | - Audio Files: `.ogg` (e.g., `sheath.ogg`, `theme.ogg`, `jumppp11.ogg`)                        |
|                              | - Scene Files: `.unity`, `.scene` (e.g., `temp_scene.unity`, `duoscene.unity`)                |
| **AI & Behavior**            | - State Machines & Animations for NPCs (e.g., zombies, slimes, turtleshells)                   |
|                              | - Sensing routines (e.g., `sensesomethingroutine_slime_anim.fbx`)                            |
|                              | - Attack & Damage Animations (e.g., `@attack 01.fbx`, `@take damage.fbx`)                     |
|                              | - Pathfinding & Movement: Likely using Unity NavMesh or custom movement scripts               |
| **Animation & Visual Effects**| - Character & Enemy Animations (`walk`, `idle`, `attack`, `hurt`, `die`)                     |
|                              | - Particle Effects & Effects Prefabs (e.g., `plasmaexplosioneffect.prefab`)                   |
|                              | - Lighting & Reflection: Lightmaps (`lightmap-2_comp_light.exr`)                              |
| **Game Mechanics & Features**  | - Multiple enemy types with distinct behaviors (zombies, slimes, turtleshells)               |
|                              | - Player controls with animations and sound effects (e.g., jump, attack, hurt)               |
|                              | - Environmental assets like chests, torches, and scene objects                                |
| **Sound & Music**            | - Background music (`theme.ogg`) and sound effects (`fireball.prefab`, `sheath.ogg`)        |
| **Scene & Level Design**       | - Multiple scene files for different levels or scenes (`temp_scene.unity`, `duoscene.unity`) |
| **Dependencies & External Assets** | - PSDs, textures, 3D models, animations, sound files, prefabs, controllers, and scene files  |
